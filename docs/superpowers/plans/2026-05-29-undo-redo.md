## Task 4 — JS：新增 undo() 與 redo() 函式

**預計時間：** 3 分鐘

### 步驟

- [ ] 緊接在修改後的 `setState()` 函式結尾（`}`）之後，插入以下兩個函式：

```js
    function undo() {
      if (historyStack.undos.length === 0) return;
      historyStack.redos.push(snapshotState());
      const prev = historyStack.undos.pop();
      Object.assign(state, prev);
      if (!state.days.find(d => d.id === state.activeDayId)) {
        state.activeDayId = state.days[0]?.id ?? null;
      }
      save();
      render();
    }

    function redo() {
      if (historyStack.redos.length === 0) return;
      historyStack.undos.push(snapshotState());
      if (historyStack.undos.length > MAX_HISTORY) historyStack.undos.shift();
      const next = historyStack.redos.pop();
      Object.assign(state, next);
      if (!state.days.find(d => d.id === state.activeDayId)) {
        state.activeDayId = state.days[0]?.id ?? null;
      }
      save();
      render();
    }

```

## 邊界條件備忘

| 情境 | 處理方式 |
|------|----------|
| undo stack 為空 | `undo()` early return；按鈕 disabled |
| redo stack 為空 | `redo()` early return；按鈕 disabled |
| 執行新操作 | `historyStack.redos = []` 清空 redo stack |
| stack 達 50 筆 | `undos.shift()` 移除最舊一筆 |
| 頁面重整 | historyStack 為記憶體物件，自然清空 |
| activeDayId 指向已刪除天數 | undo/redo 執行後與 setState 相同邏輯修復 |
| input 欄位 Ctrl+Z | `isInput` 判斷 early return，交給瀏覽器原生處理 |
| importJSON 繞過 setState | 在 `setActiveTrip` 前手動 push snapshot |
| switch-trip / new-trip-in-list | 這兩個 action 直接操作 `state` 變數而非呼叫 `setState`，屬跨行程切換，不入 history（維持現狀） |
